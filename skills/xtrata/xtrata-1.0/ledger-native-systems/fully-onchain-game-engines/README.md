# Fully On-Chain Game Engines

Game engines can be expressed as reusable graph components: rulesets, physics parameters, maps, asset packs, characters, AI routines, and score systems. A specific game becomes an instance of shared engine parts.

## Focus

- make engines and games composable instead of monolithic
- preserve deterministic rules and asset provenance
- let new games inherit proven modules without forking everything
