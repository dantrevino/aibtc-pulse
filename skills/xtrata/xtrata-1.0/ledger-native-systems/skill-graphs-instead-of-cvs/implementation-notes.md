# Implementation Notes

## Minimal prototype slice

- define a profile manifest that references a small set of work artifacts
- attach contribution claims and endorsements to those artifacts
- render a profile view centered on evidence, not a text CV

## Notes for future development

- keep human-readable summaries separate from canonical evidence links
- avoid collapsing the graph into one reputation number too early
- design for many contribution types beyond software commits

## Possible first integrations

- BNS-linked identity handles for public profiles
- Xtrata artifact references for creative, technical, or research portfolios
