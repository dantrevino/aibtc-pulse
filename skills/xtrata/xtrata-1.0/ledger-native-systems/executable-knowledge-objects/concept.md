# Concept

## Concise explanation

Executable Knowledge Objects turn research outputs into dependency graphs. A paper becomes a manifest that references datasets, code commits, parameter sets, experiment runs, peer reviews, and follow-up replications so the knowledge object can be inspected and partially rerun.

## Problem being solved

Traditional publication bundles claims, evidence, and methodology into a document that is hard to verify. Datasets move, code diverges, and replication attempts are disconnected from the original artifact.

## Why ledger-native architecture helps

- claims can point directly to datasets and executable evidence
- replication and peer review become graph-linked descendants
- provenance survives across revisions and forks
- incentives can reward reproducibility, not just citation count

## Future expansion ideas

- funding pools for high-value replication targets
- field-specific execution profiles for chemistry, ML, and climate science
- public interfaces that show which claims remain weakly supported
