# Implementation Notes

## Minimal prototype slice

- start with a small cross-domain corpus linking art, research, and governance objects
- define a base artifact schema plus a few typed relationship edges
- build a simple explorer that traverses across domains instead of staying media-specific

## Notes for future development

- expect ontology design to matter as much as storage design
- keep curation explicit so institutional and community lenses remain distinguishable
- prototype federation early to avoid central index assumptions

## Possible first integrations

- Xtrata-style recursive artifact IDs as the common object anchor
- Stacks stewardship or grant contracts for community-maintained collections
