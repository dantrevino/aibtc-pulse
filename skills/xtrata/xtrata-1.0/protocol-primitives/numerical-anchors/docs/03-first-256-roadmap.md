# First 256 Inscription Roadmap

## Objective

Deliver a coherent first-256 primitive layer with clear execution order and
minimal ambiguity for future builders.

## Milestone Plan

| Milestone | Slot Window | Outcome |
| --- | --- | --- |
| M1 | 64-101 | lock binary, audio, genesis manifest, builder starter |
| M2 | 102-144 | lock recursion safety, identity baseline, graph/structure rules |
| M3 | 145-191 | lock naming and reference resolution primitives |
| M4 | 192-255 | lock token logic + reserve controlled experiment slots |
| M5 | 256 | lock canonical hash verification anchor |

## Core Anchors (Seed Set)

| Slot | Primitive | Short Purpose |
| --- | --- | --- |
| 64 | Binary Foundation | canonical binary/chunk handling |
| 72 | Cycle Primitive | timing cadence reference for recursive execution |
| 88 | Instrument Anchor | base frequency and scale map for audio systems |
| 89 | Growth Primitive | branching/expansion reference for recursion trees |
| 97 | Compression Anchor | deterministic compression/reassembly policy |
| 100 | Genesis Manifest | root registry of early constants |
| 101 | Builder Primitive | one-call bootstrap for standard dependencies |
| 108 | Naming Basis | deterministic naming/hash policy seed |
| 111 | Self Reference | canonical self-reference and loop-safe pattern |
| 128 | Chunk Boundary | canonical boundary and reconstruction limits |
| 137 | Composability Constant | symbolic optional anchor |
| 144 | Graph Root | dependency graph format reference |
| 256 | Hash Anchor | canonical verification and normalization rules |

## Roadmap Notes

- Use Audionals as the demonstration stack for slots 72 and 88.
- Keep 100 and 101 as mandatory dependencies for future templates.
- Keep 224-255 partially reserved to avoid overcommitting early governance.

## Definition Of Done (Per Primitive)

1. Spec completed using the primitive template.
2. Byte-level payload reviewed and reproducible.
3. Dependency graph documented.
4. Test recursion example produced.
5. Slot status moved to `locked`.
