# Vision And Principles

## Vision

Treat key inscription numbers as permanent recursive protocol primitives.

For Xtrata, early numbers can become "numerical gravity wells": stable IDs that
future works reference by default because they are immutable, deterministic, and
culturally recognized.

## Design Principles

1. Immutable first, convenient second.
2. Keep primitives minimal and auditable.
3. Every primitive must improve recursion composability.
4. Prefer deterministic outputs over dynamic behavior.
5. Separate symbolic anchors from critical execution anchors.
6. Minimize upgrade pressure by locking clear interfaces early.

## What A Primitive Must Provide

- A specific job (not a broad app).
- A stable interface (inputs and outputs).
- A known recursion pattern.
- Defined failure behavior.
- Clear dependency expectations.

## Success Criteria

- Builders can assemble useful systems by referencing anchor IDs directly.
- Core anchors become reused defaults across multiple apps.
- The first 256 IDs form a coherent, documented protocol layer.
