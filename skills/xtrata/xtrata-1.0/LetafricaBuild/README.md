# Let Africa Build x Xtrata

This folder is a working brief for a dedicated Let Africa Build (LAB) landing
page inside the Xtrata ecosystem.

The goal is to introduce LAB builders to Xtrata as practical Bitcoin/Stacks
infrastructure, not as a speculative NFT tool. The page should make the case
that Xtrata is a permanent, composable data layer for products that matter in
African markets: payments, credentials, business tooling, creative economies,
local knowledge systems, and durable public infrastructure.

## Folder contents

- `research-and-positioning.md`
  - distilled research on LAB and the messaging constraints that follow
- `landing-page-blueprint.md`
  - page goals, audience, information architecture, section plan, copy angles,
    and implementation notes
- `builder-idea-bank.md`
  - expanded set of product ideas and starter tracks for LAB builders using
    Xtrata on Stacks

## Recommended outcome

Use these docs to build a dedicated LAB page that:

1. Explains Xtrata in plain builder language.
2. Connects Xtrata to LAB's infrastructure-first mission.
3. Gives LAB builders concrete things they can ship quickly.
4. Shows how STX, Stacks contracts, stablecoin rails, and permanent on-chain
   storage can combine into real products.

## Core positioning

Suggested framing:

> Build permanent apps on Bitcoin-secured infrastructure.
> Use Stacks for execution and Xtrata for memory.

Short version:

- Stacks gives LAB builders programmable Bitcoin infrastructure.
- Xtrata gives them permanent storage, recursive composition, and deployable
  on-chain media/app primitives.
- Together they support products that can be owned, verified, remixed,
  monetized, and preserved.
