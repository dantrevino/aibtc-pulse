# LAB Landing Page Blueprint

## Objective

Create a dedicated landing page for Let Africa Build builders that introduces
Xtrata as practical protocol infrastructure on Stacks and gives visitors clear
ideas for what they can build.

Primary goal:

- convert curiosity into first experiments and starter builds

Secondary goals:

- position Xtrata as serious infrastructure for the LAB ecosystem
- connect Xtrata to real African product opportunities
- create a page that LAB mentors, partners, and founders can share

## Audience

- LAB academy participants
- incubator founders and technical teams
- Stacks-curious African builders
- ecosystem partners and mentors

## Page thesis

LAB is building the builder pipeline.
Xtrata can be presented as one of the core protocol primitives those builders
can use to ship durable products on Stacks.

Plain-language thesis:

If your app needs data, files, media, templates, records, or reusable modules
to stay available and verifiable, Xtrata gives you a way to put that layer
on-chain.

## LAB pillar strip

Early on the page, add a simple 3-column strip that mirrors LAB's visible
program framing:

- Learn
  - Use Xtrata to understand how permanent, composable Bitcoin-secured apps
    work on Stacks.
- Incubate
  - Turn small experiments into products with durable records, reusable assets,
    and clearer monetization paths.
- Build infrastructure
  - Create templates, datasets, archives, components, and protocol-native tools
    other builders can reuse.

## 2. Why this matters for LAB

Section heading:

- Why LAB builders should care

Message:

LAB is not about short-term crypto hype. It is about helping African builders
create useful infrastructure. Xtrata fits that mission by making application
state, records, media, templates, and reusable logic durable by default.

Suggested 4-card grid:

- Permanent
  - Publish data and media that do not depend on a fragile backend.
- Verifiable
  - Store records and content in a way that can be reconstructed and checked.
- Composable
  - Build on previous work instead of starting from zero each time.
- Monetizable
  - Combine ownership, payments, and reusable assets into viable products.

## 3. What Xtrata actually is

Section heading:

- The protocol in plain English

Suggested explanation:

Xtrata is an on-chain data layer for Stacks. It lets builders inscribe files
and structured content in chunks, reconstruct them deterministically, and use
them as permanent building blocks inside apps, collections, archives, tools,
and media systems.

Suggested visual:

- a simple stack diagram
  - Bitcoin: settlement/security
  - Stacks: smart contracts and app logic
  - Xtrata: permanent data and reusable components
  - LAB builders: apps, startups, and local infrastructure

## 4. What LAB builders can build

Recommended categories:

Each category card should include:

- Academy
- Incubator

## 5. Starter paths

Section heading:

- Choose your build path

Recommended three-tier structure:

- Beginner
  - on-chain profile, directory, or knowledge post
- Intermediate
  - a paid content product, API, or creator storefront
- Advanced
  - a fully on-chain app, recursive media system, or shared component library

Each path should have:

- skill level
- estimated build scope
- likely Xtrata primitives used
- likely Stacks primitives used

## 7. Builder workflow

Section heading:

- How it fits together

Suggested 4-step flow:

1. Build app logic on Stacks.
2. Store assets, records, or reusable modules with Xtrata.
3. Compose or fork previous on-chain work.
4. Launch, monetize, and preserve the result.

This section should make the protocol feel usable, not abstract.

## 8. Trust and credibility

Section heading:

- Why this is more than a demo

Suggested proof elements:

- production-ready SDK
- chunked uploads and deterministic reconstruction
- SIP-009 compatibility
- existing first-party app and collection tooling
- recursive app/media support

If real live examples are available at launch, include them here.

## Recommended copy direction

### Tone

- technically serious
- optimistic
- grounded
- non-speculative

### Vocabulary to lean into

- infrastructure
- permanence
- composition
- builder pipelines
- verifiable data
- reusable modules
- durable products
- local relevance

### Vocabulary to avoid

- moon
- alpha
- degen
- yield-first framing
- NFT collectibles as the default story

## Visual direction

The page should not look like generic purple-on-black crypto marketing.

Better direction:

- warm neutrals with strong accent color
- editorial typography plus technical UI accents
- maps, grids, system diagrams, and product cards
- visual cues that suggest archives, networks, ledgers, and build systems

If it is implemented in the existing Xtrata app:

- preserve the current design system and spacing constraints
- use a dedicated section rather than forcing a full visual reset

## Suggested content modules to build later

- live examples carousel
- filtered project idea explorer
- "choose your builder track" selector
- ecosystem resource links
- FAQ for first-time Stacks builders

## Implementation notes

If this page is built inside the existing React app:

- keep the content mostly static and cache-friendly
- treat live examples as optional enhancement, not a launch blocker
- reuse existing section patterns where possible
- prefer a small number of clear CTAs over dense documentation
