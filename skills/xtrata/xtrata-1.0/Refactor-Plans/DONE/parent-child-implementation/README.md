# Parent-Child Implementation Handoff Pack

Purpose: full implementation plan for parent-child relationships in minting and viewing, aligned with the live `xtrata-v1.1.1` contract and current app constraints.

This folder is intended to be sufficient for a new assistant to implement without rediscovery.

## Files

1. `01-scope-and-contract-model.md`
Core model, scope boundaries, non-goals, and contract-level truth.

2. `02-implementation-plan.md`
Phased execution plan with concrete file touchpoints and acceptance criteria.

3. `03-test-plan.md`
Required app and contract tests, including new test cases.

4. `04-execution-checklist.md`
Step-by-step runbook for implementation and verification.

5. `05-context-map.md`
Direct navigation index of contract, app, and test touchpoints.

6. `06-full-implementation-checklist.md`
Authoritative implementation tracker with per-phase tasks, gates, and verification.

## Quick Start

1. Read `docs/app-reference.md` and `docs/assumptions.md`.
2. Read `01-scope-and-contract-model.md` in this folder.
3. Open `06-full-implementation-checklist.md` and use it as the active tracker.
4. Execute phases in `02-implementation-plan.md` in order.
5. Validate with `03-test-plan.md`.
6. Close with `04-execution-checklist.md`.
