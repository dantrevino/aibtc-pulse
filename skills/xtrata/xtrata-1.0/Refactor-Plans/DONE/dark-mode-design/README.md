# Dark Mode Design Implementation Pack

Purpose: implementation-ready plan to add a safe, streamlined theme toggle so public and admin experiences can switch between the current light style and a more modern dark style.

This pack is designed so a new assistant can execute the work without rediscovery.

## Documents

1. `01-current-state-and-gaps.md`
Current architecture, styling reality, and constraints.

2. `02-target-ux-and-theme-model.md`
Target UX, toggle behavior, and theme system model.

3. `03-implementation-plan.md`
Phased implementation plan with concrete file touchpoints.

4. `04-token-migration-spec.md`
Token design and migration strategy for safe CSS conversion.

5. `05-test-and-validation-plan.md`
Unit/manual validation strategy and acceptance criteria.

6. `06-rollout-checklist.md`
Execution checklist for safe delivery and rollback readiness.

7. `07-context-map.md`
Code navigation index for theme-related work.

## Quick Start

1. Read `docs/app-reference.md`.
2. Read `01-current-state-and-gaps.md`.
3. Implement phases in `03-implementation-plan.md`.
4. Validate with `05-test-and-validation-plan.md`.
5. Complete signoff with `06-rollout-checklist.md`.
