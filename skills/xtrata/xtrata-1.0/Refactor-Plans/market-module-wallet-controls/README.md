# Market Module + Wallet Controls Implementation Pack

Purpose: implementation-ready plan to improve the market and wallet UX so users can list, cancel, and transfer directly while viewing their own inscriptions in wallet mode.

This pack is written so a new assistant can execute without rediscovery.

## Documents

1. `01-current-state-and-gaps.md`
Current behavior, architecture reality, and concrete UX gaps.

2. `02-target-ux-and-flow.md`
Target product behavior and user flows for wallet + market modules.

3. `03-implementation-plan.md`
Phased implementation plan with file-level touchpoints.

4. `04-simplification-and-safety-optimizations.md`
Plan optimizations that reduce risk, call volume, and complexity.

5. `05-test-and-validation-plan.md`
Unit/contract/manual test plan and acceptance criteria.

6. `06-rollout-checklist.md`
Execution runbook for safe delivery.

7. `07-context-map.md`
Navigation map of relevant files and current responsibilities.

## Quick Start

1. Read `docs/app-reference.md`.
2. Read `01-current-state-and-gaps.md`.
3. Implement phases in `03-implementation-plan.md`.
4. Validate using `05-test-and-validation-plan.md`.
