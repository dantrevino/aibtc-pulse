# FOSS Hack 2026 Strategy

## Event snapshot
- Window observed: 2026-02-20 to 2026-03-31 (hybrid).
- Prize pool observed: approximately INR 500,000.
- Primary emphasis: open-source contributions and community value.
- Confidence: high.

## Recommended entry
Build **Xtrata Open Toolkit Release** as the primary submission.

## Why this is the best fit
- FOSS-focused judging favors high-quality open-source artifacts.
- Xtrata can win by shipping reusable tooling, docs, and tests rather than only
  demo UX.
- This also compounds value beyond the event.

## Product concept
Release a developer toolkit bundle:
- CLI for chunk reconstruction and deterministic verification.
- SDK helpers for read-only integration paths.
- End-to-end reference example and test fixtures.

## 60-second pitch
Xtrata Open Toolkit removes the hardest integration steps for permanent
on-chain media workflows. Instead of every team rebuilding chunk logic and
verification paths, developers get audited patterns, reproducible tests, and a
clean integration starter they can use immediately.

## Hackathon scope
Must-have:
- Open-source repo updates with clear module boundaries.
- Installation and quickstart docs.
- Test suite proving deterministic reconstruction paths.

Stretch:
- Plug-in architecture for additional chains/viewers.
- Maintainer onboarding docs for external contributors.

## Submission checklist
- Keep license and contribution guide explicit.
- Include changelog and migration notes.
- Highlight community reusability over one-off demo polish.
