# Xtrata Launch Campaign Plan

## Goal
- Build broad brand awareness and clear understanding of Xtrata.
- Convert builders and creators into active minting and integrations.
- Establish "Xtrata = on-chain memory layer" as the core mental model.

## Core audiences
1) Builders / Devs
   - Motivation: APIs, recursion, deterministic reconstruction, performance.
   - Message: Build on-chain apps/media that are permanent and modular.
2) Artists / Musicians / Creators
   - Motivation: permanence, ownership, new creative formats.
   - Attribution guaranteed, single source of truth.
   - Simplified, efficient, transparent, unified, on-chain verifiable rights management.
   - Message: Mint media that lives forever on-chain, is owned by the rights holder, and references other works and other rights holders in an unbreakable merkle tree chain of creativity and rights.
3) Collectors / Curators / Galleries
   - Motivation: authenticity, provenance, verifiable dependencies.
   - Message: True on-chain artifacts, not just metadata links.
4) Marketplaces / Platforms
   - Motivation: compatibility, richer inventory, differentiation.
   - Message: SIP-009 compatible NFTs with richer data and recursion.
5) General crypto audience
   - Motivation: innovation, Bitcoin L2, permanence.
   - Message: Xtrata is the memory layer for Bitcoin.

## Messaging pillars
1) Permanence: large on-chain data, deterministic reconstruction.
2) Recursion: inscriptions reference inscriptions (modular apps/media).
3) Compatibility: SIP-009 works with existing wallets/marketplaces.
4) Scale and efficiency: chunked uploads, batch reads, cheaper access.
5) Real use cases: apps, sound engines, datasets, recursive media.

## Primary CTA
- "Explore the viewer"
- "Mint your first inscription"
- "Read the developer guide"

## Campaign phases (6 weeks)

### Phase 1 - Awareness (Weeks 1-2)
- Objectives: reach + clear positioning
- Outputs:
  - Daily X posts + 2 threads/week
  - 3 short-form demo videos
  - 1 long-form explainer post

### Phase 2 - Education (Weeks 3-4)
- Objectives: deeper understanding + developer confidence
- Outputs:
  - 2 tutorials (video or written)
  - 2 live sessions (AMA / demo)
  - 2 case studies / showcases

### Phase 3 - Adoption (Weeks 5-6)
- Objectives: creator and partner activation
- Outputs:
  - 3-5 featured creator drops
  - 1 marketplace integration spotlight
  - 1 recap post with metrics and highlights

## Channel mix
- X/Twitter: primary daily channel
- YouTube: long-form tutorials + Shorts
- TikTok/Reels: short visuals and demos
- Discord/Farcaster/Telegram: community engagement
- Reddit: long-form updates and AMAs
- Medium/Mirror: deep-dive articles

## Success metrics
- Awareness: impressions, reach, unique profile visits
- Engagement: saves, shares, comments, watch time
- Conversion: mints, docs traffic, integration requests

## Risks + mitigations
- API rate limits: use caching and batch reads, throttle requests.
- Complex concepts: use simple visual diagrams and short explainer clips.
- Limited attention: front-load short punchy content, then link to detail.

## Core positioning statement
"Xtrata is the on-chain memory layer for Bitcoin: a permanent, recursive data
platform that powers modular media and applications on Stacks."
