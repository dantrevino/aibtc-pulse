## Observed optimisation pressure points

1. Large monolithic UI files increase maintenance and render risk.
2. Single large CSS file suggests style-scope fragmentation opportunity.
3. Large JS entry chunk indicates route/module-level splitting opportunity.
4. Manage portal repeats collection types and API-fetch patterns across panels.
5. App and SDK overlap in protocol logic; opportunity to reuse shared primitives.

## Non-goals for first optimisation pass

- No contract behavior changes.
- No mint flow order changes.
- No wallet signing semantics changes.
