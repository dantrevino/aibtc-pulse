# Stage 0 - Game Review Plan

## Objective
Create a factual baseline of the current game so future changes improve engagement without breaking pacing, readability, or score trust.

## Review Areas
1. Core Loop Quality
- Time-to-fun (first meaningful decision).
- Session shape (early, mid, late game intensity).
- Death fairness (avoidable vs unavoidable).

2. Progression & Rewards
- Upgrade cadence (how often and how impactful).
- Reward consistency by wave.
- Dead zones where player gets no meaningful change.

3. Narrative & Worldbuilding
- Current story clarity (what is happening and why).
- Emotional pacing between waves.
- Opportunities for faction lore, character stakes, and sector identity.

4. Build & Difficulty Health
- Enemy density curves by wave.
- Damage spikes and survivability windows.
- Weapon uptime and crowd-control coverage.

## Metrics to Capture
- Average run duration by skill tier.
- Death cause distribution (% by enemy/archetype).
- Upgrade offer frequency (seconds/waves between offers).
- Upgrade pick rate per option.
- Median time to first major power spike.
- Score growth rate by wave.

## Playtest Matrix
- 3 skill groups: novice, regular, expert.
- 5 runs per group per build.
- Capture: frustration points, confusion moments, excitement moments, quit moments.

## Deliverables
1. Baseline metrics snapshot.
2. Top 10 friction points.
3. Top 10 “double down” strengths.
4. Priority recommendations feeding Stage 1 roadmap.
