# Games Output Rules

This file governs the production-test game outputs in `recursive-apps/21-arcade/games/`.

## Purpose of This Folder
- `games/` is the runtime output folder loaded by the arcade launcher.
- Source development should happen in per-game workspaces (for example `game01_astro_blaster-v2/`), not directly in this folder.
- Each runtime slot should have a paired workspace at `recursive-apps/21-arcade/gameNN_slug-vX/` with its own `AGENTS.md`.

## Naming and Versioning
- Base output pattern: `gameNN_slug.js`
- Versioned output pattern: `gameNN_slug-vX.js`
- Example:
  - `game01_astro_blaster.js`
  - `game01_astro_blaster-v2.js`
- Default progression after `v2` is decimal minor:
  - `v2 -> v2.1 -> v2.2 -> v2.3`
  - only jump to a new major when explicitly requested
- Build default:
  - each new workspace build should create the next decimal versioned runtime file
  - repeated builds should continue incrementing decimal minors (`v2.1`, `v2.2`, `v2.3`, ...)
- Helper:
  - `npm run arcade:next-version -- --game game01_astro_blaster`
  - explicit major: `npm run arcade:next-version -- --game game01_astro_blaster --version 3`

## Loader and Manifest Behavior
- `index.html` loads `games/latest-manifest.js` and `lib/game-loader.js`.
- `lib/game-loader.js` resolves the highest available `-v` file for each `gameNN_slug` slot.
- `games/latest-manifest.js` is generated by `npm run arcade:games:manifest`.
- `main.js` displays the resolved game version in decimal form on each tile (`v1.0`, `v2.0`, `v2.1`, ...).

## Revert Strategy
- To revert a game version, delete the unwanted newer file (for example `game01_astro_blaster-v2.3.js`).
- Regenerate manifest: `npm run arcade:games:manifest`.
- On next load, the loader falls back to the highest remaining version.

## Promotion Checklist (Workspace -> games/)
1. Build from workspace script into `games/` output.
2. Run workspace tests.
3. Run strict strategy review gate for the game:
   - `npm run arcade:strategy:review -- --game gameNN_slug --strict`
4. Run parser checks for generated file.
5. Regenerate manifest.
6. Verify game starts and exits correctly in arcade launcher.
7. Verify score flow still uses guarded `shared.highScores` path.
8. If score submit/wallet code changed, run strict wallet diagnostics before promotion:
   - `npm run arcade:wallet:diag`

## Do Not
- Do not hand-edit `games/latest-manifest.js`.
- Do not add non-game artifacts to this folder.
- Do not bypass workspace tests when promoting a new version.
- Do not create versioned runtime files without a matching workspace folder and workspace-level `AGENTS.md`.
